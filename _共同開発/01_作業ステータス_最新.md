## 申し送り事項（引き渡し先 = AGO / 菅野社長）

### 重要
- データはすべてブラウザの localStorage に保存される
  - バックアップ: マスターパネル →「全データ書き出し」→ .json ファイルを定期保管
  - 復元: マスターパネル →「データ読み込み」→ .json ファイルを選択
- ブラウザのキャッシュ・localStorage をクリアするとデータが消える（定期バックアップ必須）
- api.php / PHP サーバーは不要（ブラウザで index.html を直接開くだけで動作）
- VAPID 鍵は config.js に記載 → 機密扱い・GitHub に push しない
- `github.com/nihonneon2025/ago-system-demo` は公開リポジトリ → 実スタッフ名・実顧客名をコミットしない

### スタッフ名の変更方法
1. `config.js` をテキストエディタで開く
2. `users` セクションの `name: 'スタッフ01'` を実名に変更
3. `masterPin: 'kanno0'` を任意の PIN に変更（推奨）
4. 保存してブラウザをリロード

### マスターモードの入り方
- 画面左上の「AGO」ロゴを5回連続タップ → PINコード入力

## 既知の課題

1. localStorage はブラウザごと・端末ごとに独立する（複数端末での同期は将来課題）
2. 写真データ（base64）が増えると localStorage 容量を圧迫する可能性（5〜10MB上限）
3. VAPID 鍵の管理場所が config.js → GitHub 公開リポジトリへのコミット禁止
4. ブラウザのプライベートモード・キャッシュクリアでデータが消えるリスク
